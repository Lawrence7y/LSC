# 无畏契约回合持续分析设计

## 目标

在用户主动选择“无畏契约/回合切割”模式后，对游戏直播与赛事解说直播识别完整的交战回合：从屏障解除开始，到本回合结算或下一回合买枪阶段开始结束。每个已确认回合要立即加入切片列表，并为所有已选择且已同步的直播间分别导出到其指定目录。

## 范围

- 仅改变无畏契约回合切割模式；其他直播类型和其他分析模式保持现有行为。
- 复用现有 OCR、音频回合检测、持续分析、同步时间映射、切片列表和导出队列。
- 不增加模型、训练流程或第三方依赖。

## 回合边界

### 开始

以 HUD 从买枪阶段切换到交战阶段的视觉状态为准，即屏障解除后的第一帧。买枪期和屏障解除前的准备画面不应落入切片。

### 结束

以本回合胜负结算画面或下一回合买枪状态的视觉状态为准。回合结束音效只用于校验和 OCR 不可用时的辅助判断，不能单独使片段具备自动导出资格。

### 确认规则

自动入列和自动导出只接受同时具备可信起点与终点的回合。边界不完整的最新回合保持“等待回合结束”状态；录制停止时仍未闭合的回合标记为未确认且不导出。该保守策略优先避免把相邻两回合合并成一个切片。

## 检测策略

`round_detector` 继续使用音频 RMS 和回合结束音效，但在无畏契约专用模式中以现有 OCR 回合状态标记作为主信号：

1. 识别买枪状态到交战状态的切换，得到切片起点。
2. 识别胜负结算或下一回合买枪状态，得到切片终点。
3. 用音频能量和结束音效校验边界；OCR 缺失时可以继续等待，不产生自动导出片段。
4. 每次连续分析只扫描新增录制区间，并保留一个固定的回看窗口以闭合跨扫描周期的最新回合。
5. 已确认回合以主房间的回合时间窗口作为去重键，重复扫描不得再次入列或导出。

这使每次扫描的工作量随新增录制时长变化，而不是随整场直播长度增长；扫描完成后释放临时音频和 OCR 帧数据，避免内存随直播时长累积。

## 多房间入列与导出

一个主房间已确认回合产生后，后端使用现有同步映射计算每个已选择、已同步目标房间的本地时间范围。每个房间生成独立切片条目，并立即推送到前端列表。

切片条目依次经历 `待导出`、`导出中`、`已导出` 或 `导出失败`。导出复用现有受控队列，避免并发导出抢占录制、OCR 和 FFmpeg 资源。每个房间按其已有的导出设置写入指定目录；单个房间失败不能阻止其他房间继续导出。

## 进度展示

持续分析状态通过现有 WebSocket 状态链路推送，前端常驻显示：

- 已录制时长和已分析时长；
- 当前阶段：等待新录制、识别回合、等待回合结束、导出中、收尾、完成或异常；
- 已确认回合数量、切片数量、成功导出数量和失败数量；
- 当前回合或当前导出任务的局部进度；
- 最新错误原因。

直播没有固定终点，因此进度以“已分析时长 / 当前已录制时长”表达当前追赶程度，不能将持续录制误表示为全局 100%。

## 错误处理

- OCR 暂时不可用：继续等待新的可识别帧，不自动以纯音频结果导出。
- 音频不可用：只要 OCR 起止边界完整，仍允许确认；状态中记录音频校验不可用。
- 同步映射或目标录制文件无效：该房间标记导出失败，其他房间继续处理。
- 导出失败：保留切片和失败原因，允许沿用现有手动导出能力重试。

## 验收与测试

1. 模拟 OCR 标记验证切片从屏障解除开始，在结算或下一回合买枪状态结束。
2. 只有起点或只有终点的回合不进入自动导出路径。
3. 同一回合重复出现在相邻扫描窗口时只产生一组切片。
4. 一组已确认回合为每个已选择且已同步房间各生成一条切片和一个导出任务。
5. 状态事件准确报告分析时长、等待闭合、导出中、成功、失败和收尾状态。
6. 非无畏契约模式不进入专用检测与自动导出路径。
