# LSC 产品评估技能

自动化执行LSC项目的产品评估报告，从用户体验、功能完整性、易用性、性能表现等多个维度进行评估。

## 使用方法

```bash
# 全面评估
python -m mimocode skill run product-review --full

# 快速评估
python -m mimocode skill run product-review --quick

# 评估特定功能
python -m mimocode skill run product-review --feature "multi-room"
```

## 评估维度

### 1. 用户体验 (UX)
- 界面布局合理性
- 交互流程顺畅度
- 视觉设计一致性
- 错误提示友好度
- 键盘快捷键支持

### 2. 功能完整性
- 核心功能实现状态
- 辅助功能完备性
- 边界情况处理
- 错误恢复机制
- 数据持久化

### 3. 易用性
- 学习曲线
- 操作步骤数量
- 默认配置合理性
- 帮助文档完整性
- 国际化支持

### 4. 性能表现
- 启动速度
- 响应时间
- 内存占用
- CPU使用率
- 磁盘IO

### 5. 代码质量
- 架构设计
- 代码可读性
- 测试覆盖率
- 错误处理
- 日志记录

## 评估流程

1. **功能清单** - 列出所有功能点
2. **逐项评估** - 按维度评估每个功能
3. **问题识别** - 发现关键问题和改进点
4. **优先级排序** - 按P0/P1/P2/P3分类
5. **生成报告** - 输出结构化评估报告

## 输出格式

```markdown
## 产品评估报告

### 执行摘要
- 评估时间: YYYY-MM-DD
- 评估范围: 功能/UX/性能
- 总体评分: X/10

### 功能清单
| 功能 | 状态 | 评分 | 问题数 |
|------|------|------|--------|
| 录制 | ✅ | 8/10 | 2 |
| 导出 | ✅ | 7/10 | 3 |

### 关键问题
#### P0 (必须修复)
1. 问题描述 - 影响范围 - 建议修复方案

#### P1 (应当修复)
1. 问题描述

#### P2 (可以改进)
1. 问题描述

### 改进建议
1. 短期改进 (1周内)
2. 中期改进 (1个月内)
3. 长期改进 (3个月内)

### 测试建议
- 需要补充的测试用例
- 自动化测试建议
```

## 注意事项

- 评估前确保程序可以正常启动
- 使用真实用户场景进行测试
- 记录具体的操作步骤和截图
- 问题描述要具体到文件路径和行号
- 修复建议要可执行
