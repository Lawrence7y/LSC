# LSC 直播切片系统 — UI/UX 改造方案

## 改造总览

| 模块 | 改动范围 | 优先级 |
|---|---|---|
| 时间线/播放器 | 大改 — 重新设计 | P0 |
| 仪表盘 | 精简 — 去掉导航卡片，保留状态+历史 | P1 |
| 右侧面板 | 优化 — 录制设置专业化，去重复 | P1 |
| 核心路径引导 | 优化 — 状态反馈+引导流程 | P2 |

---

## 一、时间线/播放器模块大改

### 1.1 底部 ControlBar 时间线（主时间线）

**现状**：40px 细线 + 圆点播放头，无刻度尺，像玩具。

**目标**：OBS + 专业标记风格，高度约 80-100px。

**改造内容**：

#### a) 时间码标尺（Ruler）
- **启用** `_draw_time_ticks`（当前是死代码），将其集成到 `paintEvent`
- 显示主刻度 + 次刻度 + 时间标签（HH:MM:SS）
- 自适应间隔：根据可见时长自动调整（1s/5s/10s/30s/1min/5min）
- 标尺区域高度约 20px，位于时间线顶部

#### b) 波形轨道区
- **单轨道**显示：背景轨道 + 已录制进度填充 + 入出点选区高亮
- 轨道高度约 20-24px，比现在粗，有明确的视觉边界
- 已录制部分用主题色填充，未录制部分用暗色背景
- 选区（入点→出点）用半透明主题色高亮 + 两侧竖线手柄

#### c) 播放头（Playhead）
- 改为**磁性三角形播放头**（类似 Premiere/DaVinci），不是圆点
- 三角形朝下，指向轨道位置，上方显示时间码
- 拖动时实时显示时间码预览
- 播放头顶部有竖线贯穿整个时间线区域

#### d) 标记点（Markers）
- 录制中按 `I` 设入点、`O` 设出点
- 入点标记为绿色三角形（在轨道上方）
- 出点标记为红色三角形（在轨道上方）
- 入出点之间的选区用半透明色块填充
- 已添加到切片列表的片段在轨道上显示为独立的色块条

#### e) 快捷键
- `I`：设入点（当前播放位置）
- `O`：设出点（当前播放位置）
- `X`：清除当前选区
- `Space`：播放/暂停
- `←/→`：后退/前进 5s
- `Shift+←/→`：后退/前进 1s（微调）
- `M`：在当前位置添加标记点（备选，如果需要纯标记功能）

### 1.2 ControlBar 按钮重新分组

**现状**：所有按钮平铺一行，无分组逻辑。

**目标**：分三组，中间用分隔线隔开。

```
[ 播放控制组 ]  |  [ 选区操作组 ]  |  [ 导出/视图组 ]
```

- **播放控制组**：后退 5s / 播放暂停 / 前进 5s / 时间码显示
- **选区操作组**：入点按钮 / 出点按钮 / 清除选区 / 添加到切片列表
- **导出/视图组**：导出选区 / 全屏

时间码显示改为**单行显示**（`00:12:34`），不需要 `/ 00:00:00` 格式（因为直播录制时长是实时增长的，没有固定总时长）。

### 1.3 卡片内迷你时间线（保留但优化）

- 保留 `_MiniTimeline`，但视觉上更紧凑
- 用途：快速浏览该房间的录制进度，拖拽定位
- 高度保持 32px，不显示刻度尺，只显示进度条 + 播放头
- 与底部 ControlBar 时间线**单向同步**：拖动卡片内时间线会更新底部 ControlBar

### 1.4 关键文件

| 文件 | 改动 |
|---|---|
| `lsc/gui/components/timeline.py` | 重写 `paintEvent`，启用刻度尺，改为三角形播放头 |
| `lsc/gui/components/control_bar.py` | 按钮分组，时间码格式调整 |
| `lsc/gui/pages/multi_room/page.py` | 时间线信号连接调整 |
| `lsc/gui/theme.py` | 新增时间线相关样式变量 |

## 二、仪表盘精简

### 2.1 删除的内容
- ❌ 4 张统计卡片（录制任务、在线直播间、待导出切片、存储使用）
- ❌ 4 张快捷操作卡片（开始录制、查看切片、管理直播间、应用设置）

### 2.2 保留/新增的内容
- ✅ **房间状态概览**：以紧凑列表形式显示所有已添加房间的状态（主播名、平台、状态、时长）
- ✅ **最近录制历史**：显示最近的录制记录，点击可自动创建该房间并跳转到工作台
- ✅ **存储使用**：保留存储统计，但放在底部不显眼位置

### 2.3 最近录制历史的交互
- 每条记录显示：主播名、平台、录制时长、录制时间、文件路径
- 点击记录 → 自动在工作台添加该 URL 的房间 → 跳转到工作台 → 选中该房间
- 如果该房间已存在，直接跳转并选中

### 2.4 关键文件

| 文件 | 改动 |
|---|---|
| `lsc/gui/pages/dashboard.py` | 大幅重写，去掉统计卡和操作卡，改为状态列表+历史列表 |
| `main.py` | `_refresh_dashboard` 方法适配新数据结构 |

## 三、右侧面板录制设置专业化

### 3.1 现状问题
- 录制设置（画质/编码器/CRF/码率）UI 是用 `ChipGroup` 实现的简单选择器
- 看起来不够专业，参数展示不直观

### 3.2 改造方向
- **画质预设**：保留 ChipGroup 三档，但增加说明文字（如"原画: 不重编码，直接拷贝流"）
- **编码器**：改为下拉框或更专业的选择器，显示完整编码器名称（如 "H.264 (NVENC) - NVIDIA GPU 加速"）
- **编码参数**：
  - CRF 模式：显示滑块 + 数值输入 + 质量说明（如 "18=高质量 23=默认 28=小体积"）
  - 码率模式：显示数值输入 + 单位选择 + 建议值提示
- **输出目录**：显示完整路径 + 已用空间 + 打开文件夹按钮

### 3.3 关键文件

| 文件 | 改动 |
|---|---|
| `lsc/gui/pages/multi_room/page.py` | `_build_record_settings_card` 方法重写 |
| `lsc/gui/components/widgets.py` | 可能需要新增专业参数控件（滑块+数值输入组合） |

## 四、核心路径引导优化

### 4.1 现状
用户打开后需要自己摸索：添加 URL → 连接 → 预览 → 录制 → 标记 → 导出

### 4.2 优化方案
- **空状态引导**：工作台没有房间时，显示引导提示（"粘贴直播间链接开始"），不是空白
- **状态反馈**：每个操作都有明确的视觉反馈（Toast 通知、状态栏更新、卡片状态变化）
- **操作提示**：连接成功后提示"可以开始录制"，录制开始后提示"按 I/O 标记精彩片段"
- **快捷键提示**：ControlBar 旁边的快捷键说明（可折叠）

### 4.3 关键文件

| 文件 | 改动 |
|---|---|
| `lsc/gui/pages/multi_room/page.py` | 空状态引导、操作提示 |
| `lsc/gui/components/room_card.py` | 卡片状态视觉反馈 |

## 六、验证方案

1. **视觉验证**：截图对比改造前后的深色/浅色主题效果
2. **功能验证**：
   - 录制中按 I/O 设入出点 → 时间线显示选区
   - 添加到切片列表 → 列表显示片段
   - 导出切片 → 文件正确生成
   - 仪表盘点击历史记录 → 自动创建房间并跳转
3. **快捷键验证**：I/O/X/Space/←/→ 全部可用
4. **主题验证**：深色和浅色主题下所有新增组件样式正确
