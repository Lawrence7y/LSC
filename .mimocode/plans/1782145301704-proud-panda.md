审查时间：2026-06-23

## 一、关键 Bug（3 个，会导致运行时崩溃）

### Bug 1：选区循环试听功能崩溃
- **文件**：`lsc/gui/multi_room/manager.py:750, 765`
- **问题**：`start_range_loop()` 和 `stop_range_loop()` 引用 `self._preview_widgets`，但 `MultiRoomManager` 类中从未定义该属性。正确写法应通过 `self.get_room(room_id).preview_widget` 获取。
- **影响**：用户在多房间工作台点击"试听选区"按钮时，程序抛出 `AttributeError` 崩溃。
- **修复**：将 `self._preview_widgets.get(room_id)` 改为 `self.get_room(room_id).preview_widget`

### Bug 2：导出模块常量定义在函数体内
- **文件**：`lsc/exporter/clip.py:42-43`
- **问题**：`_DEFAULT_AUDIO_BITRATE = "128k"` 和 `_DEFAULT_VSCALE = "scale=1080:1920"` 定义在 `parse_ffmpeg_progress_line()` 函数体内部（第 41 行后缺少空行分隔）。每次调用该函数都会重新执行赋值，且这两个常量在函数外无法访问。
- **影响**：虽然功能上暂时无害（Python 会正常执行赋值），但代码意图不清晰，且如果其他模块需要引用这些常量会失败。
- **修复**：将两个常量移到函数外部，作为模块级变量。

### Bug 3：录制页 `_log` 未导入
- **文件**：`lsc/gui/pages/record.py:1110-1143`
- **问题**：`_on_url_parsed()` 方法中使用了 `_log.debug(...)`，但 `record.py` 文件顶部从未导入 `_log`。
- **影响**：当重连 token 不匹配时，代码尝试调用 `_log.debug()` 会抛出 `NameError`，导致录制页崩溃。
- **修复**：在文件顶部添加 `from lsc import get_logger; _log = get_logger(__name__)`

## 四、错误处理缺口（8 处）

### 4.1 `print()` 代替 `logging`（3 处）
- **文件**：`cookie_helper.py:123, 187, 190-204`
- **问题**：使用 `print()` 输出日志，违反项目日志规范。
- **修复**：改为 `_log.info()` / `_log.warning()`。

### 4.2 异常被完全吞掉（5 处）
- **文件**：
  - `douyin.py:107-108` — `_get_user_live_room()` 中 `except Exception: pass`
  - `douyin.py:113-114` — 同上
  - `douyin.py:137-138` — `_extract_room_from_page()` 中 `except Exception: pass`
  - `manager.py:390` — `_load_json_file()` 中 `except Exception: return None`
  - `recording_controller.py:184-191` — NVENC 检测后台线程 `except Exception` 不记日志
- **修复**：每个 `except` 块添加 `_log.debug()` 或 `_log.warning()`。

### 4.3 生产代码使用 `assert`
- **文件**：`exporter/clip.py:236`
- **问题**：`assert proc.stdout is not None` 在 `-O` 优化模式下会被跳过。
- **修复**：改为 `if proc.stdout is None: return`。

## 五、线程安全问题（4 处）

### 5.1 Douyin 模块缓存非线程安全
- **文件**：`lsc/platforms/douyin.py:34, 190-205`
- **问题**：`_cached_module` 是 `ClassVar[ModuleType | None]`，`_load_script_module()` 非线程安全地读写它。
- **修复**：添加 `threading.Lock` 保护。

### 5.2 NVENC 缓存非线程安全
- **文件**：`lsc/gui/pages/recording_controller.py:172-207`
- **问题**：`_nvenc_cache` 和 `_nvenc_checking` 是类变量，在主线程读取、后台线程写入，无锁保护。
- **修复**：添加 `threading.Lock` 保护。

### 5.3 批量录制状态不一致风险
- **文件**：`lsc/gui/multi_room/manager.py:109-115`
- **问题**：`_BatchRecordWorker.run()` 在后台线程写入 `room.is_recording` 和 `room.record_output_path`，UI 线程可能读到中间态。
- **建议**：当前 GIL 保护下暂无实际问题，但复合操作应通过信号通知 UI 更新。

### 5.4 文件大小更新线程写入
- **文件**：`lsc/gui/multi_room/manager.py:1165`
- **问题**：`SizeUpdateRunnable.run()` 在 QThreadPool 线程写入 `room.record_size_mb`。
- **建议**：同上，当前安全但依赖 GIL 隐式假设。

## 六、资源泄漏（3 处）

### 6.1 异步刷新 Worker 永不清理
- **文件**：`lsc/gui/multi_room/manager.py:864-868`
- **问题**：`refresh_stream_url_async()` 创建的 `_RefreshWorker` 存储在 `self._connect_workers[f"_refresh_{room_id}"]` 中，但 `_cancel_connect_worker()` 只按 `room_id` 查找，不会清理 `_refresh_` 前缀的 worker。
- **修复**：在 `remove_room()` 中也清理 `_refresh_` 前缀的 worker。

### 6.2 缩略图线程池永不关闭
- **文件**：`lsc/exporter/clip.py:63-71`
- **问题**：`_thumbnail_executor` 是类级别的 `ThreadPoolExecutor`，惰性创建但从不 `shutdown()`。
- **修复**：添加 atexit hook 或在程序退出时 shutdown。

### 6.3 NVENC 检测后台线程
- **文件**：`lsc/gui/pages/recording_controller.py:182-193`
- **问题**：`start_nvenc_check()` 创建的 `threading.Thread` 是 daemon 线程，程序退出时会被强制终止，但如果有未完成的 FFprobe 调用可能留下孤儿进程。
- **建议**：当前可接受，因为是 daemon 线程。

## 七、UX 问题（6 处）

### 7.1 录制页"连接"按钮状态误导
- **文件**：`lsc/gui/pages/record.py:1089-1105`

- **问题**：`_analyze_btn.setEnabled(False)` 时没有设置 tooltip 解释原因。
- **修复**：添加 tooltip 如 "请先完成录制"。

### 7.4 录制页连接失败无重试入口
- **文件**：`lsc/gui/pages/record.py:1138-1152`

### 7.6 CRF 范围提示与滑块不一致
- **文件**：`widgets.py:220`（滑块 0-51）vs `settings.py:124`（提示"范围 18-28"）
- **问题**：用户可以设置 CRF=0（无损）或 CRF=51（极差），但设置页提示范围是 18-28。

## 十、架构建议（长期改进）

### 10.1 Toast 系统解耦
- **当前**：`multi_room.py` 通过 `import main` 跨模块调用 `_main.show_toast()`
- **建议**：通过构造函数注入或 Qt 信号传递 Toast 管理器引用

### 10.2 录制参数统一
- **当前**：`recording_controller.py` 手动拼接 FFmpeg 参数，绕过 `ExportProfile`
- **建议**：录制和导出共用 `ExportProfile` 生成 FFmpeg 参数

### 10.3 页面导航改为枚举
- **当前**：`PAGE_MAP = {"dashboard": 0, "workbench": 1, ...}` 使用魔法数字
- **建议**：改为枚举或直接存储页面引用

### 10.4 主题切换性能优化
- **当前**：`_refresh_app_style()` 遍历 `app.allWidgets()` 逐个 polish
- **建议**：只对可见页面的 widget 做 polish，隐藏页面延迟处理

### 10.5 心跳优化
- **当前**：`_on_global_tick()` 每秒对所有房间调用 `_update_card_timeline()`
- **建议**：只更新可见卡片和选中卡片的时间线
