## 发现的 Bug 与修复

### Bug 1: 前端缺失 enable_preview_response 错误处理
- **影响**：后端返回任何错误（FFmpeg不可用、无流URL等），前端静默忽略，用户点击后无反馈
- **修复**：在 `useWebSocket.ts` 添加 `enable_preview_response` 和 `request_mse_init_response` handler
- **文件**：`lsc-electron/src/hooks/useWebSocket.ts`
- **后端配合**：`_handle_mse_preview` 所有错误返回现在包含 `room_id`

### Bug 2: VideoPreview 无超时机制
- **影响**：FFmpeg 无法产出 init 段时，前端永久显示"加载中"
- **修复**：添加 15 秒超时，超时后自动通知后端关闭预览并显示错误提示
- **文件**：`lsc-electron/src/components/VideoPreview.tsx`

### Bug 3: request_mse_init 无重试上限
- **影响**：init 段未就绪时反复重试，可能无限循环
- **修复**：添加 5 次重试上限 + 递增延迟 (1s/2s/3s/4s/5s)
- **文件**：`lsc-electron/src/hooks/useWebSocket.ts`

### Bug 4: FFmpeg 命令缺少连接超时
- **影响**：直播流URL失效时FFmpeg长时间挂起
- **修复**：添加 `-timeout 10000000` (10s)、`-rw_timeout 15000000` (15s)、`-reconnect` 选项
- **文件**：`lsc/core/services/mse_streamer.py`
- **连带修复**：HLS `-live_start_index` 和 scale filter 的插入逻辑适配新参数位置

### Bug 5: mse_error 未同步清除 preview_enabled
- **影响**：MSE 错误发生时前端状态不一致
- **修复**：`mse_error` handler 同步设置 `preview_enabled: false`
- **文件**：`lsc-electron/src/hooks/useWebSocket.ts`
